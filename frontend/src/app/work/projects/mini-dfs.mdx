## Overview

MiniDfs is a lightweight distributed file system built from scratch in Java. It mimics the core functionalities of large-scale distributed systems like HDFS, providing a reliable way to store and retrieve files across multiple machines.

## Key Features

- **Automated Replication**: Ensures data durability by maintaining multiple copies of files across different nodes.
- **Fault Tolerance**: Automatically detects node failures and redistributes data to maintain the desired replication factor.
- **Centralized Master Node**: A master node that tracks file locations and manages metadata for the entire cluster.
- **Simple API**: Easy-to-use client API for uploading, downloading, and deleting files within the distributed cluster.

## Technologies Used

- **Java**
- **Networking (Sockets)**
- **Multithreading**
- **Serialization**

## Impact

Demonstrated deep understanding of distributed systems principles, including consensus, availability, and consistency. The project serves as a foundation for understanding how cloud storage providers manage massive amounts of data reliably.
